# SIM-on-FHIR

**Santeon CarePlan Implementation Guide** — beschrijft hoe het Santeon Informatie Model (SIM) wordt omgezet naar gestandaardiseerde HL7 FHIR R4-profielen.

## Publisher

[Santeon](https://www.santeon.nl) — FHIR versie: R4 (4.0.1)

---

## Getting started (Windows)

### 1. Clone the repository

```powershell
git clone https://github.com/SanteonNL/sim-on-fhir.git
cd sim-on-fhir
```

> **Line endings note:** The repo contains both `.bat` (Windows) and `.sh` (Unix) scripts. To avoid Git converting line endings and breaking the shell scripts, set this before cloning:
> ```powershell
> git config --global core.autocrlf input
> ```

### 2. Install prerequisites

| Tool | Purpose | Install |
|------|---------|---------|
| **Java** (JRE 11+) | Runs the HL7 FHIR IG Publisher | [adoptium.net](https://adoptium.net) |
| **Ruby + Devkit** | Required by the IG Publisher for Jekyll templating | [rubyinstaller.org](https://rubyinstaller.org/downloads/) — pick **Ruby+Devkit** |
| **Jekyll** | Processes IG templates into HTML | `gem install jekyll` (after Ruby) |
| **Python 3** | SQL mapping generator + local dev server | [python.org](https://www.python.org) — on Windows use `py -3` |
| **HL7 IG Publisher** (`publisher.jar`) | Builds the IG from FSH sources | Run `_updatePublisher.bat` (downloads ~100 MB) |

Verify each tool:
```powershell
java -version
ruby --version
jekyll --version
py --version
```

### 3. Install Jekyll on Windows

After installing Ruby via RubyInstaller, open a new PowerShell and run:

```powershell
ridk install 3      # installs MSYS2 + MINGW toolchain (needed for native gems)
ridk enable         # activates the toolchain in the current session
gem install jekyll
```

**If `gem install jekyll` fails due to a corporate proxy** (rubygems.org blocked), use MSYS2's package manager instead. Open the **MSYS2 UCRT64** terminal from the Start menu and run:

```bash
pacman -S mingw-w64-ucrt-x86_64-ruby
gem install jekyll bundler
```

Then add MSYS2 to your user PATH (no admin rights needed):

```powershell
[System.Environment]::SetEnvironmentVariable("PATH", $env:PATH + ";C:\msys64\ucrt64\bin", "User")
```

Close and reopen PowerShell, then verify with `jekyll --version`.

### 4. Download the IG Publisher

```bat
_updatePublisher.bat
```

Downloads `publisher.jar` (~100 MB) into `input-cache/`. Run this once on first setup, and again when you want to upgrade.

## Project Structure

```
sim-on-fhir/
├── input/
│   ├── fsh/
│   │   └── SanteonCarePlan.fsh         # FHIR profile (FSH source)
│   └── static/
│       ├── queries/
│       │   ├── luscii_careplan.sql      # LUSCII source query
│       │   └── pj_careplan.sql          # PJ source query
│       └── generate_fhir_sql_mappings.py  # Mapping generator script
├── input-cache/
│   └── publisher.jar                   # IG Publisher (not committed)
├── fsh-generated/                      # Auto-generated by publisher (do not edit)
├── output/                             # Built IG output (do not edit)
└── sushi-config.yaml                   # IG configuration
```

Key SQL source tables (LUSCII):
- `luscii_patients` — patient data
- `luscii_patientsprogramshistory` — enrollment timeline
- `luscii_programs` — program/category info
- `luscii_usersStatusChangeReasons` — status change tracking

## When you change a profile

### 1. Edit the FSH profile

Open [input/fsh/SanteonCarePlan.fsh](input/fsh/SanteonCarePlan.fsh) and make your changes.

### 2. Update SQL queries (if the data mapping changed)

Edit the relevant SQL file(s) in [input/static/queries/](input/static/queries/):
- `luscii_careplan.sql` — LUSCII data source
- `pj_careplan.sql` — PJ data source

### 3. Regenerate FSH mappings from SQL

Run the mapping generator to sync the `^mapping` statements in the FSH profile:

```bash
cd input/static
py generate_fhir_sql_mappings.py
```

This reads all `.sql` files in `queries/` and updates the mapping blocks in `SanteonCarePlan.fsh`.

### 4. Build the IG

From the repo root:

```bat
_genonce.bat
```

Build takes 1–3 minutes. Output goes to `output/`.

### 5. Preview locally

The IG pages use `fetch()`, so you need an HTTP server (not `file://`):

```bash
cd output
py -m http.server 8080
```

Then open `http://localhost:8080`.

## Updating the IG Publisher

To download the latest version of `publisher.jar`:

```bat
_updatePublisher.bat
```

This replaces `input-cache/publisher.jar` with the latest release from GitHub (~100 MB).
